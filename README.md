# bookmarks-collector-ai

一个基于 Plasmo 的浏览器扩展，用来辅助书签分类、页面知识抓取和用户确认式整理。

## 插件功能

- **当前页抓取**：自动读取页面标题、域名、`meta description`、标题层级和正文片段。
- **选中文本收集**：在 Popup 中可把当前页面的选中文字加入待整理内容。
- **页面区域框选**：在页面侧边栏里开启框选模式，点击页面元素即可抓取区域文本。
- **本地推荐**：根据当前页面内容、手动标签、备注和现有书签结构，优先给出可复用的文件夹建议。
- **AI 增强推荐**：配置 OpenAI 兼容接口后，会在本地推荐基础上调用模型补充更智能的分类结果。
- **确认后再执行**：插件不会自动改动书签结构，必须由用户确认后才会创建或移动书签。
- **历史书签整理**：在管理页中批量分析最近书签，勾选后可一次性应用推荐结果。
- **知识记录沉淀**：可把页面摘要、标签、备注、目标路径和推荐来源保存到本地知识库。
- **本地备份导出**：导出设置、知识记录、书签文件夹索引和抓取草稿快照为 JSON。

## 页面说明

- **Popup**：用于抓取当前页面、输入备注/标签、生成推荐并执行收藏。
- **Options/管理页**：用于配置模型、Prompt、策略开关，并查看历史书签批量整理。
- **页面侧边栏**：在网页内提供抓取面板、选中文本收集、元素框选和快捷入口。

## 配置项

- **接口基地址**：支持 OpenAI 兼容的 `baseUrl`。
- **模型名称**：填写用于生成推荐的模型名。
- **API Key**：仅在触发 AI 推荐时发送页面摘要到模型接口。
- **Prompt 模板**：可自定义系统提示词与用户模板。
- **策略开关**：可控制是否允许新建文件夹、是否优先复用已有结构、是否保存知识记录。

## 使用流程

1. 打开任意网页，选中一段文字，或直接使用页面侧边栏抓取内容。
2. 点击扩展 `Popup`，确认页面信息、补充标签或备注。
3. 生成收藏夹推荐，选择一个目标文件夹或新建建议。
4. 确认后执行写入，书签才会被创建或移动。
5. 如需启用 AI 推荐，到管理页配置兼容接口参数。

## 开发

```bash
pnpm dev
```

加载 `build/chrome-mv3-dev` 到浏览器即可调试。

## 构建

```bash
pnpm build
```

## 项目结构

```text
.
├── src/background.ts          # 扩展后台：消息路由、书签写入、导出、历史整理
├── src/contents/page-capture.ts # 页面抓取脚本与侧边栏 UI
├── src/popup.tsx             # 当前页推荐与确认入口
├── src/options.tsx           # 模型、策略与历史整理管理页
└── src/sdk
    ├── bookmarks.ts          # 书签树读取与应用推荐
    ├── client.ts             # 前端调用 SDK
    ├── constants.ts          # 默认配置与提示词模板
    ├── folder-recommender.ts # 本地启发式推荐引擎
    ├── provider.ts           # AI 推荐调用封装
    ├── storage.ts            # chrome.storage 封装
    ├── text.ts               # 文本处理工具
    └── types.ts              # 统一类型定义
```

## 设计取舍

- **保留用户确认权**：推荐不等于自动执行。
- **默认可离线工作**：未配置模型时也能使用本地启发式推荐。
- **统一数据抽象**：便于后续继续扩展搜索、导入导出和知识库能力。

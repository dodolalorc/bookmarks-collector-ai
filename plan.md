# 书签AI分类工具设计方案

一、背景

随着浏览器书签数量的增加，用户面临着管理和查找书签的挑战。传统的手动分类方法效率低下，且容易导致书签混乱。因此，设计一个基于AI的书签分类工具，可以帮助用户更高效地管理书签，同时保持用户对分类结果的控制权。

二、目标

1. 提高书签分类的效率，减少用户的手动操作。
2. 保持用户对分类结果的控制权，避免AI自动分类导致的不可控结果。
3. 提供一个浏览器插件，浏览器插件提供页面内容抓取方式，用户可以选择网页中需要的关键内容，作为本页面的一份知识内容，作为书签的标签，方便后续搜索和分类。
4. 支持新书签的快速分类和历史书签的批量整理，提升用户体验。
5. 提供一个简单易用的界面，支持新书签的快速分类和历史书签的批量整理。


# 设计思路

目标不是 AI 完全自动归类。
而是降低决策成本，同时保留用户控制权。

设计原则：
1. 自动推荐，而不是直接替代
2. 用户始终拥有确认权
3. 批量整理前，先确认收藏夹结构
4. 不做不可逆的自动行为

核心取舍在于：

- AI 自动化可以提升效率
- 但收藏夹结构是长期资产，必须可控，可恢复
	
三、交互设计
1. 新书签流程

流程设计为：
    1. 用户点击收藏
    2. 弹出 toast
    3. 分析页面内容
        3.1 用户可以使用抓取工具选择页面中的关键内容，作为书签的标签，方便后续搜索和分类。
        3.2 AI 根据页面内容和用户选择的标签，推荐一个或多个分类文件夹
        3.3 如果内容和当前的已有分类结构不匹配，AI 也可以推荐创建新的分类文件夹
    4. 推荐文件夹
    5. 用户确认后执行移动，更新用户的标签知识库

需要注意的是：
若有分类错误，就会在长期使用中污染收藏夹结构
	
2. 历史书签整理

旧书签整理采用两阶段设计：

第一阶段：生成推荐的分类结构
第二阶段：用户确认结构后，再分批迁移

这样做的原因是：
- 分类结构本身是一种系统设计
- 不应在未经确认的情况下自动重排

3. 基础管理功能

没有追求复杂能力，只提供必要支持：
- 🔎 搜索
- 支持书签的导入导出、手动或自动备份
- 支持书签的批量删除和移动

五、小总结

这种提效的工具，还是需要保留人的决策，来避免 AI 导致的不可控结果。
设计也非常需要自己在日常使用中，去观察问题的卡点在什么地方，真正要开发的内容可能很少。


# 技术实现
1. 浏览器插件开发

需要开发一个浏览器插件，支持主流浏览器（如 Chrome、Firefox）。插件需要具备以下功能：

- 书签管理界面：提供一个用户友好的界面，显示书签
- 页面内容抓取工具：允许用户选择网页中的关键内容，作为书签的标签
- AI 推荐系统：分析页面内容和用户选择的标签，推荐分类文件夹
- 分类确认界面：在用户确认分类之前，显示推荐的分类结果，并允许用户进行修改和确认 

使用的技术栈：
- 前端：React，用于构建用户界面，并与浏览器插件 API 进行交互
    - 使用 Plasmo 框架搭建浏览器扩展 SDK，https://plasmo.happyfe.com/
    - 使用 自定义开发的组件
    - tailwindcss 用于快速构建样式
    - biomejs 用于代码格式化和 linting
    - simple-git-hooks 用于 git hooks 管理
- 后端：Node.js

2. AI 模型选择
需要有添加常用模型的api key配置界面
- 预设人设和prompt 模板，用户可以选择适合的模型和模板进行分类推荐
- 支持用户自定义 prompt 模板，以适应不同的分类需求

3. 数据存储
- 使用浏览器的本地存储（如 IndexedDB）来存储书签数据和用户的分类结构
- 定期备份书签数据，提供导出功能，以防止数据丢失

4. 安全和隐私
- 确保用户数据的安全，避免未经授权的访问
- 提供隐私政策，明确说明数据的使用和存储方式

5. 测试和优化
- 进行充分的测试，确保插件在不同浏览器和操作系统上的兼容性
- 收集用户反馈，持续优化分类算法和用户界面
